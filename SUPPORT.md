# Support

Use [GitHub Issues](https://github.com/wat3rsh3d/codex-tools/issues) for reproducible defects, installation problems, documentation corrections, and feature proposals.

Include:

- the plugin and skill name;
- the invocation used;
- the relevant Codex surface and operating system;
- expected and observed behavior;
- validation output or a minimal synthetic reproduction; and
- no secrets, private project content, credentials, or personal data.

For Skill Maintenance reports, prefer a synthetic Codex-home fixture or redact local paths, project names, memory entries, and usage details. Do not attach raw session content or an unredacted local inventory report to a public issue.

Security-sensitive reports should avoid public issue details. Open a minimal issue requesting a private contact path without including the sensitive material.
