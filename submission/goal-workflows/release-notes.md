# Release notes

## 1.0.0 - Initial submission

Goal Workflows introduces two complementary Codex skills:

- `gp` creates evidence-grounded continuation prompts for fresh tasks in milestone, open-ended, or hint-defined modes.
- `gp-relay` coordinates a recurring chain of verified, non-overlapping successor-task handoffs using GP Relay Protocol v3. Each started successor can become the predecessor for the next handoff.

The submission includes final runtime trees, explicit invocation contracts, deterministic contract validation, five positive reviewer cases, three negative reviewer cases, starter prompts, public policy documents, and no MCP server or external authentication requirement.
