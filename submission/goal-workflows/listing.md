## Long description

Goal Workflows helps Codex projects move into fresh task contexts without losing verified intent. Use `gp` to create a concise, paste-ready continuation prompt grounded in current project instructions, checkpoints, and live state. Use `gp-relay` to coordinate a recurring chain of versioned, non-overlapping handoffs across visible successor tasks with exact project identity, milestone-based naming, and a verified READY/START handshake.

The workflows preserve decision boundaries, validation evidence, unresolved facts, and independent safe work while reducing historical conversation noise. A started relay successor becomes the next active segment and can relay again, so long-running work can rotate context repeatedly instead of stopping after one fresh task.
